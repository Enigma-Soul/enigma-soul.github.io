### STL 容器/函数名

> 用法模板：复制本文件，按需删改。STL 笔记重点是「会用 + 知道复杂度 + 知道坑」。

#### 〇、它属于什么（定位）

- **类别**：序列容器 / 关联容器 / 无序关联容器 / 容器适配器 / 算法函数 / 迭代器。
- **头文件**：`#include <xxx>`（写笔记时确认，`bits/stdc++.h` 全含但比赛外要写明）。
- **底层实现**：💡 例如 `vector`→动态数组、`map`→红黑树、`unordered_map`→哈希表、`priority_queue`→堆。底层决定复杂度。
- **一句话用途**：什么时候第一时间想到它？

---

#### 一、定义与声明

```cpp
// 声明方式
container<Type> name;
container<Type> name = {...};      // 初始化
container<Type> name(n, val);      // 指定大小/初值（按需）
```

- **模板参数**：每个尖括号里的参数是什么意思（如 `map<Key, Value, Compare>`）。
- **常用构造方式**：列 1~3 种最常用的。

#### 二、复杂度

| 操作 | 复杂度 | 备注 |
|------|--------|------|
| 插入 | $O(\ )$ | |
| 删除 | $O(\ )$ | |
| 查找 | $O(\ )$ | |
| 随机访问 | $O(\ )$ | 不支持就写「不支持」 |

- 💡 **复杂度陷阱**：例如 `vector` 中间插入是 $O(n)$、`unordered_map` 最坏 $O(n)$、`map::operator[]` 会**自动插入**。

| 接口 | 作用 | 返回值 | 复杂度 |

| `.insert()` | 插入 | | |

#### 五、关键点 & 注意事项

1. **迭代器失效**：💡 哪些操作会让迭代器失效？（`vector` 扩容、`erase` 后等）
2. **边界 / 返回值**：找不到时返回什么？空容器调用 `.front()/.top()` 是 UB。
3. **常见坑点**：例如 `map[key]` 的自动插入、`set` 不能改值只能删后重插、自定义比较函数的写法。
4. **性能注意**：什么时候它会慢？有没有更快的替代（如 `unordered_*`、手写）？
